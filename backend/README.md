# Finlay MERN Backend API

This is the newly refactored Node.js & Express backend designed for orchestrating AI agents, managing Hindsight memory, and interacting with Groq.

## Setup Instructions

**1. Install Dependencies**
Open your terminal inside the `backend` directory and run:
`npm install`

Dependencies included: `express`, `mongoose` (MongoDB), `cors`, `dotenv` (Environment variables), `groq-sdk` (LLM communication).

**2. Configure Environment Variables**
Copy `.env.example` to a new file named `.env`:
`cp .env.example .env` (or manually rename/create it).

Fill in the `.env` file with your actual keys:
```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/finlay
GROQ_API_KEY=your_groq_api_key_here
HINDSIGHT_API_KEY=your_hindsight_api_key_here
INSIGHT_API_KEY=your_insight_api_key_here
```

**3. Run the Backend**
For development mode (auto-restarts on file changes):
`npm run dev`

For production mode:
`npm start`

## Example API Request/Response

### Chat Endpoint
This is the main entry point to Qbit (the master agent orchestrator).

**Request:**
`POST http://localhost:5000/api/chat`
```json
{
  "userId": "user123",
  "message": "What is the sentiment on Bitcoin right now? Should I buy?"
}
```

**Expected AI Response (JSON format returned by the designated sub-agent):**
```json
{
  "sentiment": "bullish",
  "analysis": "Based on recent market trends and your history of preferring high-volatility assets, Bitcoin shows strong momentum...",
  "response": "Bitcoin is currently looking bullish. Given your portfolio goals, I recommend considering an entry."
}
```

---

### Memory Endpoint
Manually inject or retrieve memories from the Hindsight vector store context.

**Store a Memory (Request):**
`POST http://localhost:5000/api/memory`
```json
{
  "userId": "user123",
  "memoryType": "preference",
  "content": "User prefers high-risk, high-reward altcoin strategies.",
  "metadata": { "confidence": 0.95 }
}
```

**Retrieve a Memory (Request):**
`GET http://localhost:5000/api/memory?userId=user123&query=risk`

## Connecting the Frontend

1. Ensure your React frontend is running on a different port (e.g., `3000` or `5173`).
2. Inside your React app, when making an API call, point it to `http://localhost:5000/api`. 
3. *Best Practice:* In your React `.env` file, add:
   `VITE_API_URL=http://localhost:5000/api` (if using Vite)
   `REACT_APP_API_URL=http://localhost:5000/api` (if using Create React App)
4. Use `fetch` or `axios` in your React components like so:

```javascript
const response = await fetch(`${import.meta.env.VITE_API_URL}/chat`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ userId: 'user_1', message: 'Hello Qbit' })
});
const data = await response.json();
console.log(data); // Will log the JSON object generated by the agent
```
